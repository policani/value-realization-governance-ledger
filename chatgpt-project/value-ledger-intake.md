# Value Ledger Intake

## Minimum fields

- Initiative name.
- Source artifact: business case, charter, portfolio score, or sponsor decision.
- Approved value claim.
- Benefit type.
- Value hypothesis.
- Metric name.
- Baseline.
- Target.
- Actual or current value.
- Measurement source.
- Measure owner.
- Review frequency.
- Review date or period.

## Helpful fields

- Sponsor.
- Program / portfolio.
- Start date.
- Expected realization window.
- Dependency notes.
- Assumptions.
- Evidence link or reference.
- Finance validation status.
- Data quality note.

## If fields are missing

Do not guess. Use one of these labels:

- `Missing` - not provided.
- `Provisional` - supplied but not validated.
- `Proxy` - indirect measure.
- `Requires owner confirmation` - owner or accountable party unclear.
- `Requires finance confirmation` - financial result or conversion needs finance validation.
